# Technical Requirements Document

## 1. Technology Stack

Frontend:

- React
- TypeScript
- Vite
- Tailwind CSS

## 2. Project Architecture

```text
src/
  components/
  pages/
  hooks/
  utils/
  data/
docs/
```

The project is structured for clear separation of presentation, gameplay logic, persistence logic, and content.

## 3. Game Logic

The daily puzzle is selected with deterministic date-based logic tied to the South Africa time zone (`Africa/Johannesburg`).

Reference approach:

```text
index = current_date_seed % dataset_length
```

This ensures all players see the same puzzle on the same South Africa calendar day without requiring a backend service.

## 4. State Management

Application state is managed locally in React and persisted via `localStorage`.

Persisted values include:

- Current streak count
- Best streak
- Solved song history
- Last play date
- Daily completion record
- Active daily session guesses

This enables refresh-safe gameplay and simple static hosting without server-side state.

## 5. Dataset

The application uses a local JSON dataset containing approximately 30 Amapiano songs.

Each item includes:

- `song_id`
- `song_title`
- `primary_artist`
- `featured_artists`
- `year`
- `mood`
- `substyle`
- `tempo_band`
- `difficulty`
- `title_length`
- `spotify_id`
- `insight_story`

The dataset supports both gameplay and the informational "After the Drop" experience.

## 6. Audio Handling

The product does not self-host audio clips and does not proxy third-party previews.

If a `spotify_id` is present, the application renders an official Spotify embed on the result screen only. This embed is treated as a discovery and reward element, not as primary gameplay.

## 7. Deployment

The application is designed as a static front-end build and can be deployed to:

- GitHub Pages
- Vercel
- Netlify

For GitHub Pages, the build output is generated by Vite and published from the `dist/` directory through a deployment workflow.
