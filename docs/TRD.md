# Technical Requirements Document

## 1. Technology Stack

Frontend:

- React
- TypeScript
- Vite
- Tailwind CSS

Backend:

- Supabase Auth
- Supabase Postgres
- Supabase Row Level Security

## 2. Project Architecture

```text
src/
  components/
  pages/
  hooks/
  utils/
  data/
  lib/
  services/
docs/
supabase/
```

The project is structured for clear separation of presentation, gameplay logic, backend access, persistence logic, and content.

## 3. Game Logic

The daily puzzle is selected using the South Africa time zone (`Africa/Johannesburg`).

Primary production approach:

```text
challenge = daily_challenges where challenge_date = rsa_today
```

A local deterministic fallback still exists for frontend resilience if no backend challenge is configured.

## 4. State Management

Application state is managed in React with a hybrid persistence model:

- Supabase for authenticated user progress
- `localStorage` fallback for guest or offline-style play

Persisted values include:

- Current streak count
- Best streak
- Credits
- Solved song history
- Last play date
- Daily completion record
- Active daily session guesses

This enables refresh-safe gameplay, real user tracking, and a graceful local fallback when backend configuration is missing.

## 5. Dataset

The application uses:

- A local JSON starter dataset for bootstrapping
- A Supabase `songs` table for the live song library
- A Supabase `daily_challenges` table for admin-assigned puzzles

Each item includes:

- `song_id`
- `song_title`
- `primary_artist`
- `featured_artists`
- `year`
- `mood`
- `substyle`
- `tempo_band`
- `difficulty`
- `title_length`
- `spotify_id`
- `youtube_url`
- `insight_story`

The dataset supports both gameplay and the informational "After the Drop" experience.

## 6. Audio Handling

The product does not self-host audio clips and does not proxy third-party previews.

If a `spotify_id` is present, the application renders an official Spotify embed on the result screen only.

If a `youtube_url` is present, the application can render an official YouTube embed as a fallback discovery element.

Embeds are treated as post-game discovery/reward elements, not as primary gameplay.

## 7. Authentication And Authorization

Authentication is handled through Supabase email magic-link login.

Authorization rules include:

- Regular players can read public challenge data
- Players can write and read only their own attempts and results
- Admin users can manage songs and daily challenges
- Admin access is controlled by the `profiles.is_admin` flag

## 8. Deployment

The application is designed as a static front-end build with Supabase as the managed backend.

The frontend can be deployed to:

- GitHub Pages
- Vercel
- Netlify

For the current GitHub Pages setup, the build output is generated by Vite and published from the `gh-pages` branch.
